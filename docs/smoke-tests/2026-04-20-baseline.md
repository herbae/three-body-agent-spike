## Bugs surfaced and fixed in-flight

1. **`25be506` — Implementer `GH_TOKEN` missing.** `Implement with Claude` step had `ANTHROPIC_API_KEY`, `ISSUE_*`, `BASE`, `CLAUDE_MODEL` in `env:`, but no `GH_TOKEN`. The checkout step already threads `AGENT_PAT` into git for push, but `gh` CLI reads `GH_TOKEN` from env. Claude retried `gh pr create` twice (second time with `GH_TOKEN="${AGENT_PAT:-$GITHUB_TOKEN}"`) before ending the turn with a message pointing at the PR-new URL. Fix: add `GH_TOKEN: ${{ secrets.AGENT_PAT }}` to the step env.
2. **`8e2fef2` — Board-sync `pull_request` trigger commented out.** Upstream shipped `autoagent-board-sync.yml` with the `pull_request: types: [opened, closed]` trigger commented out. Re-enabled.
3. **(Minor, pending at time of this doc)** `Comment on success` step uses `gh pr list --search "$ISSUE_NUMBER"` — bare numeric search; returned no results, so no issue comment got posted. Patched alongside this doc to use `head:${AUTOAGENT_BRANCH_PREFIX}${ISSUE_NUMBER}` (same pattern the `Capture summary` step in the same file already uses).

## Pre-flight friction surfaced (not code bugs, but worth logging)

- **GitHub org vs user.** Upstream GraphQL always queries `organization(login: ...)`; the initial project was created user-owned (`herbae/projects/1`) and every board query returned `null`. Solution: created a free GitHub org `herbae-org`, recreated the project there, and transferred the repo. Making the queries owner-type-agnostic is a legitimate follow-up for the fork (candidate Task 4.5).
- **Status field option capitalization.** GitHub's default board template uses `Ready for QA` (lowercase `f`) but the config/workflows use `Ready For QA` (uppercase `F`). Case-sensitive. Had to rename the option on the board.
- **Projects V2 starter template shape.** New GitHub boards created from the default template ship with only `Todo / In Progress / Done` — the `Ready For QA` column had to be added manually.
- **Cross-owner issue pickup.** When the repo was user-owned (`herbae/...`) and the project was org-owned (`herbae-org/...`), the issue's Projects sidebar didn't list the org's project. Transferring the repo into the same org fixed this.
- **Shellcheck integration on CI.** `actionlint` on `ubuntu-latest` runs shellcheck automatically because shellcheck is preinstalled. Locally (without shellcheck) actionlint misses half the findings. Covered by commit `5ae412b` which set `SHELLCHECK_OPTS` env in the lint workflow to silence style/info noise and fixed real bugs (quoting, `$LOCAL_SHA` dead branch, jq `--arg done` SC1010).
